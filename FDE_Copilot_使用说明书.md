# FDE Copilot 使用说明书

> 一句话介绍：FDE Copilot 是你做 FDE 项目时的全程 AI 搭档，从第一次接触客户到项目收尾，帮你生成文档、写代码、记录进展、给出建议。

---

## 快速开始

### WorkBuddy 版
在任意对话中说以下任意一句话，即可激活：
```
/fde
开始FDE项目
我要去对接客户了
帮我做一个FDE项目
```
AI 会自动检测当前目录是否有项目记录：
- **有记录** → 读取上次进度，接着往下走
- **没有记录** → 进入初始化，收集项目基本信息

### 通用版
`FDE_Copilot_Universal.md` 可用于任何 AI 工具：

| 平台 | 接入方式 |
|------|---------|
| ChatGPT | Custom Instructions 或创建 GPT |
| Claude | Projects → 系统提示词 |
| Cursor | `.cursorrules` 文件 |
| GitHub Copilot | `.github/copilot-instructions.md` |
| 其他 | 对话开头粘贴或上传为上下文文件 |

## 能帮你做什么

### 📋 文档生成

| 文档 | 说什么触发 |
|------|-----------|
| 利益相关方地图 | 初始化时自动生成 |
| 访谈问题清单 | "我要去访谈客户了" |
| MVD 范围确认单 | "帮我出一个范围确认单" |
| 时间计划表 | "给我出个时间计划" |
| 每日进展消息 | "帮我写今天的进展"（然后随口说今天做了啥）|
| 技术债记录 | 自动触发（你说"先不管"时会追问并记录）|
| 演示准备清单 | "要给客户演示了" |
| 操作手册 | "项目要收尾了，帮我写操作手册" |
| 内部复盘文档 | "帮我写内部反馈文档" |

---

### 💻 Vibe Coding

你不需要写完整需求文档，口语化描述即可：

**示例对话**：
```
你：帮我写一个接收 Excel 文件、解析里面的销售数据然后展示汇总的功能

AI：你的意思是：用户上传 Excel → 系统读取指定列（哪几列？）
    → 计算汇总（求和还是平均？）→ 在页面显示结果表格，对吗？

你：对，列名是"销售额"和"日期"，按月份汇总求和

AI：[生成代码]
```

**规则**：AI 会先用一句话确认理解，你说"对"之后再写代码。

---

### 🎯 演示方法论

说"要给客户演示了"，AI 会帮你：
1. 用真实数据而非样例数据准备 Demo
2. 设计演示叙事（先演示最有价值的场景）
3. 准备离线备用方案
4. 演示后帮你记录反馈并分类

---

### 🔄 处理客户反馈

把客户反馈告诉 AI（口头描述即可），AI 会：
1. 分类：Must fix / Should fix / Defer
2. 对 Must fix 给出修改方案
3. 对超出范围的，给你一句话说辞回复客户
4. 中等变更评估对时间线的影响
5. 大变更建议重新对齐 MVD

**示例**：
```
你：客户说按钮太小了，而且他们想要一个导出 PDF 的功能

AI：
- 按钮太小 → Should fix，体验问题（30分钟内能改）
- 导出 PDF → 超出当前 MVD 范围，建议回复：
  "PDF导出功能理解，建议放到第二阶段，先确保核心功能按时交付，你觉得可以吗？"
```

---

### 🗺️ 利益相关方管理

初始化时 AI 会帮你画利益相关方地图：
- 高层决策者 → 每周汇报，用业务语言和数字
- 业务负责人 → 每天进展，用 Demo 优于文字
- IT/技术负责人 → 按需沟通，重点讲安全合规
- 一线用户 → 每次迭代后，步骤要具体

---

### 💡 随时问建议

不需要特定格式，直接问：
```
"这个功能先自动化还是手动跑一下？"
"客户突然说要加一个审批流程，我该怎么回应？"
"这段代码安全吗？"
"我感觉进度要拖了怎么办？"
"优雅实现还是快速实现？"
```

AI 会从 FDE 视角给出具体建议，不是泛泛而谈。

## 项目文件说明

### WorkBuddy 版
激活后，AI 在项目目录下创建 `.fde/` 文件夹：
```
.fde/
├── project.json        ← 项目信息 + 当前阶段
├── stakeholders.md     ← 利益相关方地图
├── mvd_confirm.md      ← MVD 确认单
├── interview_notes.md  ← 访谈记录
├── timeline.md         ← 时间计划表
├── tech_debt.md        ← 技术债记录
├── daily_log.md        ← 每日进展日志
└── demo_notes.md       ← 演示记录
```

### 通用版
- 方式A：用户在本地创建 `.fde/` 文件夹手动管理
- 方式B：AI 在对话结尾输出状态快照，下次对话开头粘贴

## 六个阶段一览

| 阶段 | 你做什么 | AI 帮你做什么 |
|------|---------|--------------|
| **0. 初始化** | 回答5个基本问题 | 生成项目文件 + 利益相关方地图 + 进场准备清单 |
| **1. 调研** | 去访谈，把结果反馈给 AI | 生成行业定制访谈问题 + 整理记录 + 输出 MVD 草稿 |
| **2. 对齐** | 和客户确认范围 | 生成 MVD 确认单 + 时间计划表 |
| **3. 执行** | 描述功能 + 反馈客户意见 | 写代码 + 处理反馈 + 需求变更管理 + 技术债 + 每日进展 |
| **3.5 演示** | 给客户做 Demo | 演示准备清单 + 演示话术 + 容错方案 + 反馈分类 |
| **4. 上线** | 逐项核对 | 生产就绪检查 + 7步部署流程 + 回滚方案 |
| **5. 收尾** | 确认价值数据 | 操作手册 + FAQ + 内部复盘文档 |

## 两个版本的区别

| 特性 | WorkBuddy 版 | 通用版 |
|------|-------------|--------|
| 状态管理 | 自动读写 `.fde/` 文件 | 用户手动管理或对话内快照 |
| 项目续接 | 自动检测并恢复 | 用户粘贴 project.json |
| 适用平台 | WorkBuddy only | ChatGPT/Claude/Cursor/任何 AI |
| 文件格式 | SKILL.md（带 frontmatter） | 纯 Markdown |
| 触发方式 | `/fde` 命令 | 系统提示词/粘贴 |
| GitHub 适配 | 需要改 | 直接可用 |

## 常见问题

**Q：跨天继续项目怎么操作？**
WorkBuddy 版：同目录下说 `/fde`，AI 自动读取 `.fde/project.json` 恢复进度。
通用版：对话开头粘贴上次的状态快照。

**Q：同时有多个 FDE 项目怎么办？**
每个项目放在不同的目录下（WorkBuddy 版），或分别保存各自的 project.json。

**Q：客户反馈很多，不知道优先做哪个？**
把所有反馈一次性告诉 AI，AI 帮你按 Must fix/Should fix/Defer 分类后按优先级排序。

**Q：代码写到一半发现方向错了怎么办？**
直接告诉 AI "方向要调整，原来想做X，现在改成Y"，AI 重新确认理解后给出新方案。

**Q：上线后客户又提新需求，算第二个项目还是继续这个？**
建议新建目录作为第二阶段（新的 `.fde/`），保持第一阶段记录完整。

**Q：要不要上 GitHub？**
通用版（`FDE_Copilot_Universal.md`）可以直接放到 GitHub，因为它是纯 Markdown、不依赖任何平台。WorkBuddy 版需要适配后才能独立使用。
